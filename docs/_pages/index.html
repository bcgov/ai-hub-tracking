Project Vision: A Shared AI Landing Zone

Why This Exists

The B.C. Government is rapidly adopting AI technologies. However, setting up secure, compliant cloud infrastructure for AI workloads allows teams to get bogged down in networking, identity management, and security compliance.

AI Hub Tracking serves as a reference implementation and reusable landing zone pattern. It solves the "blank slate" problem by providing a standardized, secure-by-default environment specifically designed for AI experimentation and production.

Problems We Solve

  • Secret Fatigue: Eliminates the need to rotate client secrets by using OIDC.
  • Network Complexity: Pre-wired VNet, Subnets, and NSGs so you don't have to be a network engineer.
  • Security Compliance: Implements "Least Privilege" and private networking by default.
  • Inconsistent Environments: Ensures Dev and Prod look identical, reducing deployment surprises.

What Can You Run Here?

This landing zone is optimized for modern AI/ML workloads that require secure compute and private data access.

Step 1: Initial Setup

Run this one-time command to create the Azure identity and trust relationship:

./initial-setup/initial-azure-setup.sh \
    -g "YOUR-RESOURCE-GROUP" \
    -n "your-app-identity" \
    -r "bcgov/your-repo" \
    -e "dev" \
    --create-storage --create-github-secrets
